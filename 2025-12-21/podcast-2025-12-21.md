Alex: First, quick framing. When people say “cut filler words, coughs, silence, noise reduction,” they usually mean programmatic audio post‑production — which is broadly machine learning plus signal processing, not necessarily the fancy generative text-to-speech stuff. API means application programming interface — a programmable way to talk to a service.
Maya: Right. So if you want to build an automated pipeline for podcasts or interviews, you want tools you can script against, and you want something that does voice activity detection (VAD), filler detection, noise reduction, silence trimming, and loudness normalization.

Alex: Tool highlights from the thread: Descript and Auphonic. Descript is great at transcript-based editing and filler-word removal — that’s why Nirant recommended it. But as Aditya discovered, Descript’s “Edit with Descript” API exists but can be gated: branded endpoint and manual approval for API access.
Maya: Auphonic is designed for programmatic post-production. It has an API that’s straightforward to call, and it’s strong on loudness normalization, noise reduction, and metadata — ideal for batch processing many episodes.

Alex: Practical alternatives to combine in a pipeline: ffmpeg for scripting audio processing; RNNoise or WebRTC noise suppression for lightweight programmatic noise reduction; iZotope RX if you want premium, manual repair; and SoX or ffmpeg’s silenceremove for silence trimming. For transcript-driven edits, Descript (for human-like filler removal) or open-source speech-to-text like Whisper for building your own cut-and-splice logic.
Maya: If you want everything in one place with less engineering, try Auphonic first. If you need more human-like, transcript-aware edits — deletions of “um” and “uh” — Descript usually does the best job, but be prepared for API access friction and branding rules.

Alex: Non-obvious takeaways. One: don’t rely on a single aggressive pass. Run an automated pass, but keep a short human review step. Overzealous noise reduction and filler removal can make speech sound unnatural.
Maya: Two: preserve the raw files and generate diffs or edit logs. If you ever need to restore a cut cough or calibrate settings, having the raw audio and timestamps saved saves headaches.
Alex: Three: combine services. For example, use Auphonic for leveling and loudness, then Descript for transcript-aware cleanup, and final ffmpeg commands for metadata and encoding. That gets you reliable automated outputs with human-quality edits where it matters.
Maya: Four: watch out for API terms. As Aditya noticed, some APIs require branding, quota review, or manual approval. If you’re building a product around an API, start that paperwork early.
